# 監視・障害対応

## ノード実行時のエラー

### バイナリが見つからない

- **原因**: `postinstall` が正常に完了していない、または `bin/ghpp` が削除された
- **対応**: `bin/ghpp` を削除して `npm install` を再実行、または Docker イメージを再ビルド

### GitHub API エラー

- **原因**: Token の権限不足、期限切れ、API レート制限
- **対応**: Credentials の Token を確認し、`repo` + `project` スコープがあることを確認する

### JSON パース失敗

- **症状**: ノード出力が `{ raw: "..." }` になる
- **原因**: ghpp のバージョン不整合、または予期しない標準出力
- **対応**: ghpp バイナリを更新する（`bin/ghpp` を削除して再インストール）

## n8n ワークフロー設計のヒント

- `ghpp promote` は冪等性がある（同じ Issue を再度昇格しない）ため、定期実行しても安全
- Cron トリガーで定期実行する場合、エラー時に通知を送るフローを組み合わせると運用しやすい
- 昇格結果の `summary.promoted` が 0 の場合は通知をスキップする分岐を入れると良い
